What you should know

Best Practices

  • Check the status of your repo before making a commit
  • Work on a branch
  • Use git mv to move a file (so it's tracked)
  • Use git rm to remove a file (so it's tracked)
  • Read your error messages CAREFULLY
  • Use var to declare new variables
  • Use git from your root repo
  • Write down a plan
  • Time-box all things
  • Check your dependencies
  • Use symbols as keys in Ruby hash

Worst Practices

  • Use numbers in column names
  • Put a repo inside another repo
  • Use 'rm' unless you are ABSOLUTELY sure of what it will do
  • Use tag id attribute to style your element
  • Be afraid to ask for help
  • Give up/Surrender
  • Streak on the 6th floor
